# Contributing

Thank you for contributing to this sandbox. This repository is intentionally lightweight, but every change should still model clean GitHub practice.

## Good Contributions

- Add a clearly named practice note.
- Improve documentation.
- Add a small workflow experiment with clear purpose.
- Fix broken links, typos, or Markdown rendering issues.

## Pull Request Checklist

- [ ] The change is small and easy to review.
- [ ] The pull request title explains the change.
- [ ] The body states why the change belongs in a sandbox.
- [ ] No private information is included.
- [ ] No empty contribution-padding commits are included.

## Branch Naming

Use short descriptive names:

```text
docs/add-practice-note-001
docs/improve-readme
ci/test-link-check
```

## Commit Style

Use clear commit messages:

```text
Add practice note for branch workflow
Improve sandbox README structure
Add link-check workflow example
```
